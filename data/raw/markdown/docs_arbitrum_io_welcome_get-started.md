# Get started with Arbitrum
Arbitrum is a suite of Ethereum scaling solutions that make it easy to build and use decentralized applications. This document provides a high-level overview of the Arbitrum suite along with onboarding guidance tailored to specific audiences.
## The Arbitrum suite[​](https://docs.arbitrum.io/get-started/overview#the-arbitrum-suite "Direct link to The Arbitrum suite")
The Arbitrum suite includes the protocols, chains, services, and SDKs that power the Arbitrum ecosystem:
Component | Description  
---|---  
[Arbitrum Rollup](https://docs.arbitrum.io/how-arbitrum-works/inside-arbitrum-nitro) | A **protocol** for scaling Ethereum smart contracts.  
[Arbitrum AnyTrust](https://docs.arbitrum.io/how-arbitrum-works/deep-dives/anytrust-protocol) | A **protocol** for scaling Ethereum smart contracts even further, with a mild trust assumption.  
[Arbitrum Nitro](https://docs.arbitrum.io/how-arbitrum-works/inside-arbitrum-nitro) | The node **software** that codifies the Rollup and AnyTrust protocols.  
[Arbitrum nodes](https://docs.arbitrum.io/run-arbitrum-node/run-full-node) |  **Machines** that run Nitro in order to service and/or interact with an Arbitrum chain.  
[Arbitrum One](https://portal.arbitrum.io/?chains=arbitrum-one) | A public Rollup **chain**.  
[Arbitrum Nova](https://portal.arbitrum.io/?chains=arbitrum-nova) | A public AnyTrust **chain**.  
[Arbitrum bridge](https://bridge.arbitrum.io/) | Lets you move `ETH` and `ERC-20` tokens between Ethereum, Arbitrum, and select Arbitrum chains.  
[Arbitrum chains](https://arbitrum.io/launch-chain) | Lets you run your own Rollup and AnyTrust chains.  
[Arbitrum Stylus](https://docs.arbitrum.io/stylus/gentle-introduction) | Lets you write EVM-compatible smart contracts in Rust and any other language that compiles to Wasm.  
## Arbitrum for users[​](https://docs.arbitrum.io/get-started/overview#arbitrum-for-users "Direct link to Arbitrum for users")
**Users** interact with Arbitrum either through the Arbitrum bridge or by using dApps that have been deployed to an Arbitrum chain.
Resource | Description  
---|---  
[Arbitrum bridge](https://bridge.arbitrum.io/) | Lets you move `ETH` and `ERC-20` tokens between Ethereum, Arbitrum, and select Arbitrum chains.  
[Arbitrum Portal](https://portal.arbitrum.io/) | A directory of dApps on Arbitrum.  
[Quickstart (bridge)](https://docs.arbitrum.io/arbitrum-bridge/quickstart) | Provides step-by-step instructions for first-time bridge users.  
## Arbitrum for developers[​](https://docs.arbitrum.io/get-started/overview#arbitrum-for-developers "Direct link to Arbitrum for developers")
**Developers** build Arbitrum dApps by deploying smart contracts to an Arbitrum chain.
Resource | Description  
---|---  
[Arbitrum Introduction](https://docs.arbitrum.io/get-started/arbitrum-introduction) | Frequently asked questions about Arbitrum's suite of scaling solutions.  
[Quickstart (Solidity)](https://docs.arbitrum.io/build-decentralized-apps/quickstart-solidity-remix) | For Web2 developers who want to deploy their first Solidity smart contract to Arbitrum.  
[Quickstart (Rust)](https://docs.arbitrum.io/stylus/quickstart) | For Web3 developers who want to deploy their first Rust smart contract to Arbitrum using Stylus.  
## Arbitrum for node runners[​](https://docs.arbitrum.io/get-started/overview#arbitrum-for-node-runners "Direct link to Arbitrum for node runners")
**Node runners** run the machines that support the Arbitrum ecosystem.
Resource | Description  
---|---  
[Run a full node](https://docs.arbitrum.io/run-arbitrum-node/run-full-node) | For node runners who want to access Arbitrum chains without having to connect to a third-party node.  
[Configure a Data Availability Committee](https://docs.arbitrum.io/run-arbitrum-node/data-availability-committees/get-started) | For Data Availability Committee members and Arbitrum chain operators who want to run a Data Availability Server.  
[Run an Arbitrum feed relay](https://docs.arbitrum.io/run-arbitrum-node/run-feed-relay) | For node runners running multiple nodes.  
[how to run an Arbitrum archive node](https://docs.arbitrum.io/run-arbitrum-node/more-types/run-archive-node) | When access to extensive historical data or advanced analytical purposes is required.
